# LifeRoadmap: актуальный отчет

## Кратко о проекте

**LifeRoadmap** — offline-first PWA для иерархического управления задачами и дорожными картами.

Проект фокусируется на трех принципах:

- **Декомпозиция целей в глубину**: любая задача может иметь подзадачи.
- **Локальная надежность**: приложение полноценно работает без интернета.
- **Приватность данных**: поддерживается E2EE и безопасная синхронизация.

## Актуальный функционал

### Управление задачами

- Древовидная структура задач с гибким перемещением веток.
- Приоритеты, дедлайны, прогресс по листовым узлам.
- Импорт/экспорт веток в JSON.

### Планирование времени

- **Регулярные задачи** (daily/weekly/monthly) на базе текущей модели `Node`.
- **Недельный таймлайн занятости** (rolling 7 days от сегодняшнего дня).
- Поддержка параллельных задач и визуального разведения пересечений в расписании.
- Для обычных задач доступно **опциональное время окончания**.
- В редакторе есть предупреждение о пересечении времени с уже запланированными задачами.

### Интеграции и уведомления

- Firebase Auth + Firestore sync.
- Telegram-уведомления (через workflow/скрипты).
- Синхронизация между вкладками браузера (`BroadcastChannel`).

## Техническая архитектура

### Данные

- Базовая сущность: `Node` (`src/types.ts`).
- Хранение — плоское (`id`, `parentId`), дерево строится в памяти.
- Локальное хранилище: IndexedDB (`src/db.ts`).
- Облачная синхронизация: `src/firebase/sync.ts`.

### Безопасность

- Шифрование на базе Web Crypto (AES-GCM).
- Два режима работы с ключами:
  - **Standard**: удобный режим с расшифровкой названий для уведомлений.
  - **Enhanced**: усиленная приватность, сервер не видит ключ.

### Конфликты и консистентность

- Стратегия синка: LWW + сравнение различий.
- Критичные проверки различий: `src/utils/syncCompare.ts`.
- Поддержка обратной совместимости при изменениях модели.

## UX и интерфейс

- Desktop + mobile сценарии, включая мобильный action drum.
- Drag-and-drop, анимации и сезонные визуальные эффекты.
- Отдельные режимы отображения дедлайнов: список, календарь, неделя.

## Процессы разработки (актуально)

- Основная среда: Cursor.
- В проект добавлены `AGENTS.md` и `.cursor/rules/`* для стабильной AI-разработки.
- Команда проверки качества:

```bash
npm run verify
```

Запускает тесты (`test:ci`) и production build.

- CI дополняется отдельной проверкой на PR/ветки (`.github/workflows/ci.yml`).
